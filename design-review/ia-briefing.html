MOMENTUS · the-moment.us · 사이트 정의서 · 2026-07-09

편집샵 IA — 쓰는 사람 관점의 구조

제공자 언어(북마크릿/확장)로 묶던 걸 버리고, "뭐 하려고 왔나"로 진열한다.

0 · 컨셉 한 줄

미니멀 갤러리 룩 + 쇼핑몰만큼 쉬운 UX + 개인 편집샵 큐레이션.
일반인이 쇼핑몰 돌듯 편하게, 그런데 아무거나 파는 잡화점이 아니라 주인 취향이 있는 편집샵. 보기는 갤러리, 쓰기는 쇼핑몰.

1 · 진열 원칙 (제일 중요)

옷을 봉제법(북마크릿/확장)으로 안 나누고 겉옷·상의·악세사리처럼 나눈다. 기준은 "이 손님이 뭘 하려고 왔나."

버림 (제공자 언어)씀 (쓰는 사람 언어)
북마크릿 / 크롬 익스텐션리서치 · 커머스 · 생산성 (내가 하려는 일)
드래그 vs 설치= 카테고리 아님. 제품 상세에 작은 배지로만("끌어놓기 / 설치")

2 · 카테고리 (진열대) — 지금 6개 제품 매핑

🔍리서치 — 잘 나가는 거 찾기
"뭐가 인기지? 레퍼런스 좀 훔치자"
인스타 인기순 정렬끌어놓기유튜브 인기순 정렬끌어놓기핀터레스트 원본 추출끌어놓기
🛒커머스 — 팔고 소싱할 때
"이 상품 재고·옵션 빨리 보자"
퀵팡 · 쿠팡 퀵보기끌어놓기 · 1만
⚡생산성 — 시간·손 아끼기
"긴 건 요약, 타이핑 대신 말로"
ChatPage · 유튜브 3초 요약설치her · 음성 입력설치
참고: 한 코너에 1개여도 편집샵은 괜찮음(작은 코너, 제품 늘면 참). 플랫폼(인스타·유튜브·쿠팡)은 2차 필터 태그로 — "인스타 쓸 때"로도 찾게. 카테고리 이름·개수는 조정 가능.

3 · 네비게이션 (GNB)

[MOMENTUS]        리서치 · 커머스 · 생산성        소개   문의
  • 로고 = 홈. 3 진열대를 바로 노출(쇼핑몰 매대 간판처럼) — 일반인이 헤매지 않게.
  • 소개 = 주인 이야기. 문의 = 우측 작게(또는 푸터).
  • "전체 보기"는 홈/컬렉션에서. 검색은 나중(제품 늘면).

4 · About 정의 — "회사소개" 아님

편집샵 주인 이야기. 강형모=이 가게를 혼자 꾸리는 사람.

  • 담을 것: 나 한 문단(1인 AI 스튜디오) · 왜 다 무료로 푸나 · 만든 것 타임라인 · 소셜/GitHub
  • 톤: 정직·개인적(회사말투 X). "이 사람 재밌네, 팬 되고 싶다"로 이어지는 층
  • 형태: 단독 페이지 /about + 홈 하단에 짧은 teaser(→더 보기)

5 · 문의 정의 — 오버스펙 금지

  • 기본: mailto:hello@the-moment.us + 인스타 DM 링크. 큰 폼·티켓 시스템 안 만듦(문의량 적음 = 오버스펙).
  • 위치: 푸터 상주 + /about 하단 + GNB 우측 작게. 별도 큰 페이지 불필요.
  • 정 필요하면 초경량 mailto 버튼 "무엇이든 물어보세요" 하나.

6 · 제품 상세 정의 (쓰는 사람 언어)

제품 이름
"이거 하면 뭐가 좋아?"  ← 한 줄 (기능 설명 아님, 결과)
[ 지금 써보기 ]         ← 끌어놓기 / 설치 (작은 배지)
이렇게 써요 · 1·2·3     ← 3스텝 + GIF 자리
자주 묻는 것            ← 질문형(AEO도 겸함)
같은 코너 다른 것        ← 귀환 갈고리(이 카테고리 이웃)
만든 사람 → /about

7 · 사이트맵 (갱신)

the-moment.us
├─ /                        홈 · 편집샵 입구
│    히어로 → 3 진열대 → 베스트(퀵팡 1만) → 소개 teaser
├─ /tools                   전체 보기
│    ├─ /tools/research      🔍 리서치 진열대
│    ├─ /tools/commerce      🛒 커머스 진열대
│    ├─ /tools/productivity  ⚡ 생산성 진열대
│    └─ /tools/[slug]        제품 상세 (예: /tools/instagram-sort)
├─ /about                   소개(주인 이야기)
├─ 문의                     mailto (페이지 아님)
└─ /legal · /privacy        재사용(이미 있음)
URL 메모: 화면 라벨은 한글(리서치·커머스·생산성), URL은 영문 SEO 슬러그. 카테고리 슬러그(research 등)는 예약어라 제품 slug와 안 겹침. 원하면 /tools/research/instagram-sort로 빵부스러기(breadcrumb) 깊이도 가능.

8 · 뒤에서 (CMS · 정적 — 요약)

  • 제품 등록 = 리포 안 플랫 파일 1개 추가 → git push → 자동 배포. DB·헤드리스 SaaS·admin 앱 안 만듦(오버스펙).
  • Astro Content Collections + 스키마 검증. 이 봇이 자동 등록도 가능.
  • Astro 정적 빌드 = "스태틱처럼 관리"(쉬움) + "LLM이 읽음"(AEO)이 한 시스템. 상세페이지가 AEO 유닛(질문형 헤딩+원자답변). 진짜 트래픽은 페이지 밖 다출처.

형이 정해줄 것

  • ① 카테고리 3개(리서치·커머스·생산성) 이름·구성 OK? 더 세분/합칠지
  • ② 제품을 뭐라 부를까 — "무료 도구" 유지 vs 다른 이름(꿀템 등)
  • ③ URL /tools/* vs 한글 카테고리 노출 방식
