# the-moment.us — 컨셉 잠금 (2026-07-09)

> 더는 드리프트 금지. 모든 디자인/카피 결정은 이 문서를 통과한다.

## 한 문장

**MOMENTUS = 제품 소비 우선. 브라우저 제품을 발견 → "지금 써보기"까지 마찰 0으로 가져가는 편집샵.**

## 우리가 아닌 것 (반례로 못 박기)

- ❌ 갤러리·포트폴리오·에이전시 쇼릴 (silent-house·Lambert류) = **콘텐츠 감상형**. 목적이 "본다".
- ❌ 무드·풍경 b-roll 히어로.
- ✅ 우리는 **제품 소비형**. 목적이 "가진다/쓴다". (Raycast·Arc·Grammarly류)

| | 콘텐츠 소비형 | **제품 소비형 (우리)** |
|---|---|---|
| 방문 목적 | 본다/감상 | **가진다/쓴다** |
| 히어로 | 무드·쇼릴 | **제품이 작동하는 화면** + "지금 설치/써보기" |
| 미디어 | 스톡 영상·풍경 | **실제 제품 UI 작동(스크린레코딩·gif·UI목업)** |
| 성공 지표 | 체류·감탄 | **설치했다·써봤다·돌아왔다** |

## 잠긴 규칙 (자동으로 따라오는 것)

1. **히어로 = 제품이 실제로 일하는 화면.** 첫 화면에서 "이게 뭘 해주는지" 3초 안에 눈으로 봄.
2. **미디어 내용 = 제품 작동.** 풍경·인물 스톡❌. 제품 UI가 자기 일 하는 장면⭕. (영상이면 제품의 영상, 이미지면 제품의 화면)
3. **"설치/드래그(써보기)" CTA는 항상 손닿는 곳.** 마찰 최소 — 로그인·결제 없음을 명시.
4. **폴리시는 유지.** 형이 좋아한 풀스크린 큰 미디어·넉넉한 여백·스크롤 리빌은 그대로 — **단 그 안을 "제품 작동"으로 채운다.** (갤러리의 미감 + 제품의 목적)
5. **제품이 주인공, 나(강형모)는 뒤.** 소개는 감동 뒤에.
6. **컴포넌트로 짓는다.** 제품 = 필드 채우기(매니페스트). 나중에 실제 스크린레코딩만 갈아끼우면 되게.
7. **콘텐츠(저널)는 보조.** 제품 소비가 1순위, 저널은 신뢰·SEO 보조.

## "영상 vs 이미지" 결론

제품 작동 = 스크린 레코딩이므로 **영상이 맞다. 단 풍경이 아니라 제품의 영상.** 지금 프로토는 실제 레코딩 대신 **CSS 제품-UI 목업(작동 애니메이션)**으로 대체 → 운영 때 실제 레코딩/gif로 교체.

## 적용 순서

1. 랜딩 히어로 → "제품 작동" 데모로 교체 (풍경 영상 폐기)
2. 각 제품 섹션 → 그 제품 UI 작동 목업 + 설치 CTA
3. 상세페이지 → hivelab식 시원한 여백 + 우측 설치 컬럼(스티키) + 컴포넌트화
