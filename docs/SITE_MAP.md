## ① 제품 리스팅 — 필터(고객 니즈)
전체 · 🆕 새로 나온(new) · ⚡ 일 빨리 끝내기(fast) · 🛒 파는 사람 도구(sell) · 🔎 콘텐츠·리서치(research) · 📚 배우고 정리(study).
- 필터는 서브메뉴처럼 보이되 클릭 동작. `?f=` 딥링크. 제품은 복수 카테고리 가능.
- 카드 → 상세. 상세 CTA는 **상태별**: 제작중=알림받기(waitlist) / 출시=설치.

## ② 저널 — 그룹 + 댓글
- **가이드·칼럼** (AX·사용법 연재, the-edit 톤: 포인트컬러·사진)
- **메이킹 로그** (제품 제작 여정 = 빌드인퍼블릭, 제품과 연결)
- 댓글 = **소셜 인증** 오픈소스: Remark42(추천·자체호스팅 소셜) / Giscus(깃허브) / Cusdis(경량)
- 레이아웃은 Mark `insights/[slug].astro` 클론(우리 소스).

## ③ 만들어드려요 (Lab) — "투표 문제"의 답
- "이거 불편해요" 제보 → 공감(upvote) → 공감순 = 제작 후보.
- **기한 약속 X** (죽은 투표·깨진 약속 방지). "볼게요, 공감 많으면 만들 확률↑".
- = 투표 + 접수 + 수요 테스트 + 커뮤니티, 한 창구.

## 🔄 관통 흐름 (살아있는 소통 허브)
제보·공감(투표) → 픽 → 제작 시작(저널 메이킹로그 + 제품 waitlist) → 출시(메일) → 리스팅 등재 → 사용·댓글 → 다음 제보.

## 제품 상태
아이디어(제보) → 제작중(메이킹로그 + waitlist 알림) → 출시(제품). 상세페이지가 상태별로 CTA 변환.

## 푸터
전 메뉴 바로가기 + 카테고리 딥링크 + 문의 + 소셜 + 법정.
